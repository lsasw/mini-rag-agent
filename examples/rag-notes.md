# RAG 学习笔记

RAG 是 Retrieval-Augmented Generation 的缩写，中文常叫检索增强生成。

一个基础 RAG 流程通常包括：

1. 加载文档
2. 切分文本
3. 为文本片段建立索引
4. 根据用户问题检索相关片段
5. 把检索结果放进 prompt
6. 调用大模型生成答案
7. 返回答案和引用来源

RAG 的价值是让模型回答时参考外部知识，从而减少幻觉，并让答案更容易追溯来源。

Agent 是在大模型回答之外加入工具调用能力。比如一个 Agent 可以先检索知识库，再调用计算器，最后把两个结果合并成回答。

LlamaIndex 也可以用来管理文档索引。常见操作包括插入文档、根据文档 ID 删除文档、更新文档，以及通过 retriever 查询相关节点。
